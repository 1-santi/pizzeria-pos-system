=====================================================
   SISTEMA DE GESTION LPM PIZZAS (v2.0)
   Instrucciones Rapidas
=====================================================

REQUISITOS:
- Windows 10 o 11
- Python 3.7 o superior (descargar de python.org)
  IMPORTANTE: Al instalar Python, marcar "Add python.exe to PATH"

COMO INICIAR:
1. Copiar toda la carpeta "proyecto de la pizzeria" a la PC
2. Configurar el modo de impresion en "config.py" (ver abajo)
3. Hacer doble clic en "run_pizzeria.bat"

CONFIGURAR IMPRESION (config.py):
- Abrir "config.py" con el Bloc de notas
- Buscar la linea OUTPUT_MODE y cambiarla:
  * OUTPUT_MODE = 'POS80'  --> Impresora termica real (en el local)
  * OUTPUT_MODE = 'PDF'    --> Genera PDF (para probar sin impresora)
  * OUTPUT_MODE = 'DEBUG'  --> Solo muestra en pantalla

CONFIGURAR IMPRESORA TERMICA:
- Ir a Configuracion > Dispositivos > Impresoras
- Clic derecho en la impresora > Propiedades
- Pestana "Compartir" > Activar "Compartir esta impresora"
- Nombre del recurso: POS80 (exacto, en mayusculas)

ARCHIVOS IMPORTANTES:
- main.py          : Punto de entrada del programa
- config.py        : Configuración del sistema (EDITAR ACÁ)
- domain/          : Modelos de datos (Product, Order, etc.)
- services/        : Lógica de negocio (pedidos, reportes, etc.)
- ui/              : Interfaz de consola (menús, pantallas)
- infra/           : Base de datos SQLite e impresora
- data/            : Carpeta que contiene la base de datos (pizzeria.db)
- tickets/         : Carpeta donde se guardan comandas, tickets y reportes
- run_pizzeria.bat : Acceso directo para iniciar el programa

NOTAS:
- Los datos se guardan en "data/pizzeria.db" (base de datos SQLite)
- NO borrar este archivo ni su carpeta, ahí está todo el historial
- El sistema NO necesita internet para funcionar
- Para más información, ver README.md

=====================================================
  Si algo no funciona, revisar:
  1. Que Python esté instalado (escribir "python --version" o "py --version" en cmd)
  2. Que la impresora esté compartida como "POS80"
  3. Que OUTPUT_MODE esté bien configurado en config.py
